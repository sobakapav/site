## <a name="anchor1" />Бизнес-задача
Компания пришла к нам с двумя ключевыми запросами:
1. Улучшить работу «умного» поиска. Он должен быть быстрее, точнее и понятнее для пользователей.
2. Сделать управление пользователями и доступами удобнее. Администраторы должны легко видеть полномочия, назначать и менять права.

## <a name="anchor2" />Дизайн-задача

Для поиска:
- провести анализ и зафиксировать ключевые сценарии;
- предложить концепты «умного» поиска, включая фильтрацию.

Для управления пользователями:
- изучить требования и текущее решение;
- проанализировать отзывы пользователей;
- разработать концепты админ-панели для выдачи прав и управления доступом.

## <a name="anchor3" />Результат
На этапе работы с административной панелью проект приостановился по внутренним причинам заказчика. Поэтому мы финализировали только первую задачу — «умный» поиск.

<em>Классическое взаимодействие: <a href="/images/portfolio/g-mini/7.png" target="_blank">строка поиска</a>, <a href="/images/portfolio/g-mini/8.png" target="_blank">Ответ ИИ в модальном окне</a>, <a href="/images/portfolio/g-mini/9.png" target="_blank">новый поиск</a></em>

Что мы сделали для «умного» поиска:
- предложили 3 концепции «умного» поиска;
- показали два варианта взаимодействия («классический» поиск и чат-бот);
- подготовили 4 формата выдачи результатов;
- добавили 8 идей-фич, повышающих удобство.

Что мы сделали для административной панели успели:
- разложить текущее флоу на 5 сценариев и 45+ макетов;
- выявить проблемы в процессе выдачи прав и доступов.

### Дополнительно

Команда дизайнеров предложила решения за рамками ТЗ. Одно из них — режим диалога с ИИ во время поиска. Заказчик отметил это как интересное направление для будущего.

## <a name="anchor4" /> Нюансы и особенности процесса

Мы работали в формате доверенного эксперимента: создавали концепты, обсуждали их на созвонах, вместе отсеивали то, что не актуально. 
Это позволило сфокусироваться на поиске новых решений, а не на преждевременной детализации.
