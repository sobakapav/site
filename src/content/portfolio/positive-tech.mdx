## <a name="anchor1" />Бизнес-задача

Разработать удобный интерфейс для нового продукта, который предлагает решение для контроля и оценки существующих проблем информационной безопасности. Продукт должен позволить пользователям не только отслеживать проблемы, но и прогнозировать выполнение показателей безопасности, обеспечивая тем самым более эффективное управление рисками.

<a href="/images/portfolio/positive-tech/1.png" target="_blank">![Интерфейс системы контроля безопасности | SobakaPav.ru](~/assets/images/portfolio/positive-tech/1.png)</a>
<emp>Дашборд</emp>

## <a name="anchor2" />Дизайн-задача

- Проанализировать функционал и архитектуру сервиса.
- Создать концептуальные решения разделов сервиса и выбрать подходящий, отвечающий требованиям MVP.
- Детализировать выбранные варианты, доработать с учетом обратной связи заказчика.
- Подготовить интерактивный прототип MVP для презентации потенциальным клиентам заказчика.

<a href="/images/portfolio/positive-tech/2.png" target="_blank">![Интерфейс системы контроля безопасности | SobakaPav.ru](~/assets/images/portfolio/positive-tech/2.png)</a>

## <a name="anchor3" />Результат

Заказчик получил кликабельные прототипы различных сценариев работы с системой, которые позволили презентовать продукт и получить заказ на разработку подобных решений.

<a href="/images/portfolio/positive-tech/3.png" target="_blank">![Интерфейс системы контроля безопасности | SobakaPav.ru](~/assets/images/portfolio/positive-tech/3.png)</a>
<em>Раздел «Задачи»</em>

Артефакты проекта:
- бизнес-требования;
- детализированный список пользовательских сценариев для создания MVP продукта;
- требования к визуалу будущего интерфейса;
- концептуальную наброски будущего интерфейса на примере двух-трех экранов в двух-трех концептуальных решениях;
- быбранный концепт в двух-трех вариантах визуального решения;
- макеты интерфейсов в Figma согласно MVP;
- библиотека используемых компонентов (UI-кит);
- интерактивный прототип по MVP, подготовленный для презентации.

## <a name="anchor4" />Особенности процесса

Процесс проектирования интерфейса был разделен на несколько этапов с активным участием заказчика. Мы использовали ежедневные короткие созвоны, чтобы постоянно сверяться и получать обратную связь, что позволило оперативно вносить корректировки в дизайн и делать его более точным с учетом потребностей потенциальных пользователей.

<a href="/images/portfolio/positive-tech/6.png" target="_blank">![Интерфейс системы контроля безопасности | SobakaPav.ru](~/assets/images/portfolio/positive-tech/6.png)</a>
<em>Сводка по группе компаний</em>

Одним из требований был строгие дедлайны, что требовало принятия решений по концептам на ходу и сокращения числа вариантов. В таких условиях ежедневные созвоны и быстрая обратная связь с заказчиком сыграли ключевую роль.
