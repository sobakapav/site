# Установим разные программы на Windows

Для тех из вас, кто пользуется Windows, процесс может более или менее отличаться от [описанного Олей П. в инструкции для Mac'а](https://github.com/sobakapav/contramotio/blob/main/%D0%A3%D1%81%D1%82%D0%B0%D0%BD%D0%BE%D0%B2%D0%B8%D0%BC%20%D1%80%D0%B0%D0%B7%D0%BD%D1%8B%D0%B5%20%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D1%8B%20%D0%BD%D0%B0%20Mac.md).

Более или менее — потому что в Windows командная строка тоже важная штука, но далеко не настолько, как в Mac или Linux. Но все-таки потренироваться ее использовать имеет смысл. Поэтому мы пойдем сложным путем, но в конце будет альтернативная инструкцию для ленивых и торопливых.

Желательно идти по шагам.

1. CLI — Command Line Interface
2. пакетный менеджер (`winget`)
3. Node.js — среда, в которой работает всё остальное
4. Build Tools (иногда нужны)
5. git — работа с git и, главное, Github
6. VS Code — редактор кода

Нельзя гарантировать, что других шагов никогда не будет.
Но эти нужны точно.

> [!WARNING]
> Не откладывайте установку программ.
> Если что-то пойдёт не так, то исправления могут занять кучу времени.
> Лучше сделать всё заранее. Как видите, этого «всего» немало.

## Открываем командную консоль

Командная строка, «терминал».
Древний, но до сих пор очень важный способ отдавать компьютеру команды и получать ответы.

*Нажмите и удерживайте кнопку Windows + нажмите R.*

В открывшемся в левом нижнем углу окошке пишем `cmd`, ввод - видим чёрное окошко со страшными белыми буквами. Теперь напишите: `dir` и нажмите Enter. Увидите список файлов.

Список файлов *где*? Напишите: `cd` и нажмите Enter. Увидите путь к папке, в которой вы сейчас находитесь. Например, автор этой инструкции сейчас видит там `C:\Users\olga>`.

Теперь вы можете запускать команды. Некоторые работают везде, некоторые — нет. Некоторые компьютер понимает, на некоторые ругается. Такая помесь «бродилки» с ChatGPT.

С этого момента в инструкциях будут встречаться плашки вроде такой:

```bash
node -v
```

Их нужно:
* копировать
* вставлять в терминал
* нажимать Enter

Вдумываться не обязательно. Обычно всё просто работает. А если не работает — пишет много букв, которые можно отдать ChatGPT.

Чтобы немного попривыкнуть, попробуйте:

```bash
echo Привет, я терминал. На вид страшный, но очень полезный.
```

Очень многие вещи делаются через CLI. Потому что GUI (кнопок) для них просто нет.

> [!WARNING]
> Это не больно! Вы просто пишете слова в окошко. Как чат. Только с компьютером.


## Проверяем наличие Winget

Это встроенный менеджер пакетов. Он устанавливает программы. Проверьте, что он есть:


```bash
winget --version
```

Если показал версию — всё хорошо. Если нет — нужно обновить Windows (серьёзно).

(Можно ввести просто `winget`: если появится простыня справочного текста, то все хорошо. Если нет — надо обновить Windows).

## Node.js — среда, в которой работает всё остальное

Сейчас будет первая установка.

```bash
winget install OpenJS.NodeJS
```

Побегут строчки — это нормально. Проверяем:

```bash
node -v
```

Должны увидеть какие-то цифры вроде:

```bash
v25.x.x
```

Это версия node. Всё хорошо, если показал. Если нет — беда, зовите на помощь, дальше идти смысла нет.

Важно: можно просить помощь у ChatGPT. Буквально скопировать ему все строчки, которые выводились во время инсталляции node. Особенно последние 15–20 строчек. Даже не поясняйте ничего, он сам всё поймёт и объяснит.

Проверьте еще две вещи:

```bash
npm -v
```

```bash
npx -v
```

Тоже появиятся цифры но другие, потому что программа другая — и номер версии у неё другой. Если цифр нет — зовите на помощь.

Кстати, многие программы можно запускать с добавкой (ключом) -v — и получать номер версии. Запомните паттерн:

```
установили → проверили через -v
```

> [!WARNING]
> Node.js ≠ Next.js. Второе слово вы скоро узнаете. Постараемся их не путать:)

## Ставим Git

Это система контроля версий. Сможете работать вместе с кем-то над одним и тем же проектом. Хранить резервные копии. Экспериментировать, не боясь что-то сломать. Хвастаться на GitHub. В общем, нужно.

Ставим:

```bash
winget install Git.Git
```

Проверяем:

```bash
git -v
```

> [!WARNING]
> `git` сложный. Но его можно делегировать ChatGPT. Главное — установите.

## VS Code — редактор кода

Это как Figma, только для программирования. Все программисты живут там.

(Кроме тех, кто живет в Idea, другом редакторе кода, как автор этой иструкции. Можете поставить ее, если вы косомолец и без трудностей не можете.)

У VS Code — ура! — есть графический интерфейс. Он вам не понравится (он никому не нравится), но он хотя бы есть.

Ставим VS Code:

```bash
winget install Microsoft.VisualStudioCode
```

После установки программа появится в меню «Пуск». Запустите её, проверьте версию командой `code -v`. Если команда не работает — перезапустите терминал.

> [!WARNING]
> На самом деле это просто редактор текста. Бесплатный и для начала удобный. В основном вы будете читать там кусочки кода, даже не писать.
